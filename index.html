Arquitectura del Sistema

El sistema está compuesto por tres capas principales que trabajan en conjunto para ofrecer una solución completa de transcripción y traducción.

Interfaz moderna con wizard de 3 pasos, temas claro/oscuro y diseño responsivo

Backend Flask

API REST con endpoints para procesamiento, manejo de errores y servicio de archivos

Python Flask

Modelos de IA

Whisper NLLB
Exposición Pública con Ngrok

El sistema puede ser expuesto públicamente usando Ngrok, permitiendo acceso desde cualquier dispositivo con internet mientras el backend se ejecuta localmente. Esto habilita un uso remoto sin necesidad de despliegue en servidores cloud.

Flujo de Trabajo Paso a Paso

El proceso completo desde que el usuario ingresa la URL hasta que recibe el archivo procesado.

1
URL del Video
El usuario ingresa la URL de YouTube. El sistema valida el enlace y prepara la descarga del video.
2
Configuración de Traducción
Selección de idioma destino (español, inglés, quechua) o solo transcripción sin traducción.
3
Selección de Modelo
Elección entre modelo rápido (600M) o de alta calidad (1.3B) para la traducción.
4
Formato de Salida
Selección entre archivo SRT de subtítulos o video MP4 con subtítulos incrustados permanentemente.
5
Procesamiento Automático
Descarga, transcripción con Whisper, traducción con NLLB y generación del archivo final.
6
Descarga del Resultado
Entrega del archivo generado y opción para procesar otro video inmediatamente.

Reglas de Traducción Automáticas
Español ↔ Inglés: Traducción directa utilizando NLLB
Español ↔ Quechua: Traducción directa con soporte nativo
Inglés → Quechua: Traducción en dos pasos (en→es→qu)
Mismo idioma: Solo transcripción sin traducción

Características Técnicas Avanzadas

Funcionalidades y optimizaciones implementadas en el sistema:

Detección automática de idioma con Whisper - no requiere especificación manual
Interfaz web responsiva con modo claro/oscuro y diseño moderno
Wizard de 3 pasos intuitivo para configuración sencilla y guiada
Gestión inteligente de memoria GPU - libera modelos cuando se cambian
Limpieza de repeticiones en traducciones para mejorar calidad del texto
Sistema de nombres único - usa sufijos _v2, _v3 para evitar sobreescritura
Múltiples métodos de descarga - yt-dlp con fallback a pytube como respaldo
API REST completa con manejo robusto de errores y CORS habilitado

Formatos de Salida Disponibles

Elige el formato que mejor se adapte a tus necesidades de uso:

Archivo SRT (Subtítulos)
Subtítulos independientes para usar en cualquier reproductor de video o plataforma
Extensión:.srt
Tamaño:Muy pequeño
Compatibilidad:Universal
Ejemplo:video_es-en.srt
Video MP4 (Subtítulos Incrustados)
Video completo con subtítulos permanentemente incrustados usando FFmpeg
Extensión:.mp4
Tamaño:Similar al original
Compatibilidad:Todo dispositivo
Ejemplo:video_es-en.mp4
Procesamiento de Video con FFmpeg

Para videos MP4 con subtítulos, se utiliza FFmpeg para incrustar permanentemente los subtítulos en el video, creando un archivo listo para reproducción en cualquier dispositivo sin necesidad de archivos adicionales. Los subtítulos quedan "quemados" en el video.
